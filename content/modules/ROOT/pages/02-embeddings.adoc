= Text Embeddings
:navtitle: Module 2: Embeddings

IMPORTANT: Before running any command in this module, make sure `LITELLM_API_BASE` and `LITELLM_API_KEY` are set in your terminal. If not, go to xref:00-setup.adoc[Setup and Credentials] and complete Step 2.

Embeddings convert text into dense numeric vectors that capture semantic meaning.
Two pieces of text that mean similar things will have vectors that are close together — even if they use different words.
This is the foundation of semantic search, RAG (Retrieval-Augmented Generation), and clustering.

== Learning objectives

By the end of this module, you'll be able to:

* Generate embeddings with curl and Python
* Compute similarity between two texts
* Understand how embeddings fit into a RAG pipeline

== Exercise 1: Generate an embedding with curl

[source,bash,subs="attributes+"]
----
curl $LITELLM_API_BASE/embeddings \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LITELLM_API_KEY" \
  -d '{
    "model": "nomic-embed-text-v1-5",
    "input": "Red Hat OpenShift is an enterprise Kubernetes platform."
  }'
----

The response contains a `data[0].embedding` field — a list of 768 floating-point numbers representing the text.

To see just the dimension count:

[source,bash]
----
curl $LITELLM_API_BASE/embeddings \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LITELLM_API_KEY" \
  -s \
  -d '{"model":"nomic-embed-text-v1-5","input":"Hello world"}' | \
  python3 -c "import sys,json; v=json.load(sys.stdin)['data'][0]['embedding']; print(f'Dimensions: {len(v)}')"
----

Expected: `Dimensions: 768`

=== Verify

✓ You receive a numeric vector with 768 dimensions.

== Exercise 2: Semantic similarity in Python

[source,python,subs="attributes+"]
----
import math
from openai import OpenAI

client = OpenAI(
    base_url="{litellm_api_base_url}",
    api_key="{litellm_virtual_key}"
)

def embed(text):
    response = client.embeddings.create(
        model="nomic-embed-text-v1-5",
        input=text
    )
    return response.data[0].embedding

def cosine_similarity(a, b):
    dot = sum(x * y for x, y in zip(a, b))
    mag_a = math.sqrt(sum(x**2 for x in a))
    mag_b = math.sqrt(sum(x**2 for x in b))
    return dot / (mag_a * mag_b)

texts = [
    "OpenShift is a Kubernetes distribution from Red Hat",
    "Kubernetes is a container orchestration system",
    "Python is a popular programming language",
]

query = "What is OpenShift?"
query_vec = embed(query)

print(f"Query: {query}\n")
for text in texts:
    vec = embed(text)
    score = cosine_similarity(query_vec, vec)
    print(f"  {score:.3f}  {text}")
----

=== Expected output

----
Query: What is OpenShift?

  0.912  OpenShift is a Kubernetes distribution from Red Hat
  0.887  Kubernetes is a container orchestration system
  0.743  Python is a popular programming language
----

=== Verify

✓ The two Kubernetes-related texts score higher than the Python text.
✓ Semantic similarity works even though the query doesn't share exact words with the documents.

== Batch embeddings

You can embed multiple texts in one API call:

[source,python,subs="attributes+"]
----
from openai import OpenAI

client = OpenAI(
    base_url="{litellm_api_base_url}",
    api_key="{litellm_virtual_key}"
)

response = client.embeddings.create(
    model="nomic-embed-text-v1-5",
    input=[
        "What is Kubernetes?",
        "How does OpenShift extend Kubernetes?",
        "What is a container?"
    ]
)

for i, item in enumerate(response.data):
    print(f"Text {i}: {len(item.embedding)} dimensions")
----
