= Module 1: Chat Models with curl
:navtitle: Module 1: Chat with curl
:source-highlighter: rouge

The LiteMaaS API is fully OpenAI-compatible. Every call uses the same format as the OpenAI API — so any existing OpenAI curl example you find will work here by changing the base URL and key.

== Learning objectives

By the end of this module, you'll be able to:

* Send a chat completion request with curl
* Stream a response token by token
* Send a multi-turn conversation with message history

== Exercise 1: Basic chat completion

Send your first request to a chat model.

[source,bash,subs="attributes+"]
----
curl $LITELLM_API_BASE/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LITELLM_API_KEY" \
  -d '{
    "model": "granite-3-2-8b-instruct",
    "messages": [
      {"role": "user", "content": "What is Red Hat OpenShift in one sentence?"}
    ],
    "max_tokens": 100
  }'
----

The response contains the model's reply in `choices[0].message.content`.

To extract just the text:

[source,bash]
----
curl $LITELLM_API_BASE/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LITELLM_API_KEY" \
  -s \
  -d '{
    "model": "granite-3-2-8b-instruct",
    "messages": [{"role": "user", "content": "What is Red Hat OpenShift in one sentence?"}],
    "max_tokens": 100
  }' | python3 -c "import sys,json; print(json.load(sys.stdin)['choices'][0]['message']['content'])"
----

=== Verify

✓ You receive a text response about OpenShift.

== Exercise 2: Streaming responses

Add `"stream": true` to receive tokens as they are generated rather than waiting for the full response.

[source,bash]
----
curl $LITELLM_API_BASE/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LITELLM_API_KEY" \
  -d '{
    "model": "granite-3-2-8b-instruct",
    "messages": [{"role": "user", "content": "List 5 benefits of containers"}],
    "stream": true
  }'
----

You will see `data: {...}` lines arriving one by one as the model generates each token.

=== Verify

✓ You see output appearing progressively, not all at once.

== Exercise 3: Multi-turn conversation

Pass previous messages in the `messages` array to give the model conversation history.

[source,bash]
----
curl $LITELLM_API_BASE/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LITELLM_API_KEY" \
  -d '{
    "model": "granite-3-2-8b-instruct",
    "messages": [
      {"role": "user", "content": "What is Kubernetes?"},
      {"role": "assistant", "content": "Kubernetes is an open-source container orchestration platform that automates deployment, scaling, and management of containerized applications."},
      {"role": "user", "content": "How does OpenShift build on top of that?"}
    ],
    "max_tokens": 200
  }'
----

The model sees the full conversation and responds in context.

=== Verify

✓ The response references Kubernetes and explains what OpenShift adds to it.

== System prompt

Use the `system` role to set instructions that apply to the whole conversation:

[source,bash]
----
curl $LITELLM_API_BASE/chat/completions \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer $LITELLM_API_KEY" \
  -d '{
    "model": "granite-3-2-8b-instruct",
    "messages": [
      {"role": "system", "content": "You are a helpful assistant that answers in exactly 2 bullet points."},
      {"role": "user", "content": "What are the main benefits of GitOps?"}
    ]
  }'
----
