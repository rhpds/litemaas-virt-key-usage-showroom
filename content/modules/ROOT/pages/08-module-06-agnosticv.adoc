= AgnosticV Integration Guide
:navtitle: AgnosticV Integration Guide
:source-highlighter: rouge

Reference for lab authors who want to add LiteMaaS virtual keys to their own AgnosticV catalogs.

== Standard catalog configuration

Add the following to your catalog's `common.yaml`:

[source,yaml]
----
# Standard AgnosticV includes — required in every catalog
#include /includes/agd-v2-mapping.yaml
#include /includes/terms-of-service.yaml
#include /includes/parameters/purpose.yaml

# LiteMaaS connection (URL + master key)
#include /includes/secrets/litemaas-master_api.yaml
# Passes requester_email and catalog_item_name to LiteLLM for usage tracking
#include /includes/parameters/litellm_metadata.yaml

# Collection requirement
requirements_content:
  collections:
    - name: "https://github.com/rhpds/rhpds.litellm_virtual_keys.git"
      type: git
      version: main

# Workloads — run litellm keys BEFORE Showroom so credentials are baked in
workloads:
  - rhpds.litellm_virtual_keys.ocp4_workload_litellm_virtual_keys

# Run on destroy to delete the key
remove_workloads:
  - rhpds.litellm_virtual_keys.ocp4_workload_litellm_virtual_keys

# Hardcode the models your lab uses
ocp4_workload_litellm_virtual_keys_models:
  - "granite-3-2-8b-instruct"
  - "nomic-embed-text-v1-5"

# Enable Docling for PDF/DOCX → Markdown conversion
ocp4_workload_litellm_virtual_keys_include_docling: false

ocp4_workload_litellm_virtual_keys_duration: "7d"
----

== Enable Docling

[source,yaml]
----
ocp4_workload_litellm_virtual_keys_include_docling: "{{ enable_docling | default(false) | bool }}"

__meta__:
  catalog:
    parameters:
      - name: enable_docling
        formLabel: Document Conversion (Docling)
        formGroup: Models
        openAPIV3Schema:
          type: boolean
          default: false
----

== Optional — model checkboxes in order form

Instead of hardcoding, let users pick models at order time:

[source,yaml]
----
ocp4_workload_litellm_virtual_keys_models: >-
  {{
    (model_granite_8b  | default(true)  | bool | ternary(['granite-3-2-8b-instruct'], [])) +
    (model_nomic_embed | default(false) | bool | ternary(['nomic-embed-text-v1-5'], [])) +
    (model_docling_258m| default(false) | bool | ternary(['deepseek-r1-distill-qwen-14b'], []))
  }}

__meta__:
  catalog:
    parameters:
      - name: model_granite_8b
        formLabel: Granite 3.2 8B Instruct
        formGroup: Models
        openAPIV3Schema:
          type: boolean
          default: true
      # repeat for each model...
----

== Multi-user workshops

[source,yaml]
----
__meta__:
  catalog:
    multiuser: true
    parameters:
      - name: num_users
        formLabel: Number of Users
        formGroup: Lab Configuration
        openAPIV3Schema:
          type: integer
          default: 1
          minimum: 1
          maximum: 30
----

TIP: `{litellm_virtual_key}` automatically resolves to each logged-in user's own key — no changes needed in your content.

== User data variables

These are injected as AsciiDoc attributes in your Showroom content.
Always add `subs="attributes+"` to code blocks that use them.

[cols="2,3",options="header"]
|===
|Variable |Value

|`litellm_api_base_url`
|Full `/v1` base URL

|`litellm_virtual_key`
|The `sk-...` virtual key

|`litellm_available_models_list`
|Comma-separated model names (for display)

|`litellm_key_duration`
|Key validity (e.g., `7d`)

|`litellm_tokenize_endpoint`
|`/v1/tokenize` URL

|`litellm_embeddings_endpoint`
|`/v1/embeddings` URL

|`litellm_docling_endpoint`
|Docling URL — *only defined* when `include_docling: true`

|`litellm_docling_model`
|Docling model name — only defined when docling is enabled
|===

== Showroom conditionals

|`litellm_has_qwen3`
|`qwen3-14b`

|`llama-scout-17b`

|Docling (use this for docling guard)
|===
