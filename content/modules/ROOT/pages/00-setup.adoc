= Setup and Credentials
:navtitle: Setup & Credentials

Before starting the exercises, set up access to your OpenShift cluster and your LiteMaaS API credentials.

== OpenShift Console

Use the *OCP Console* tab on the right to access the cluster.
Log in with the credentials below:

== LiteMaaS API credentials

== Step 1: Open the terminal

Click the *Terminal* tab on the right to open a bash session on the bastion host.

== Step 2: Set your environment variables

Run the following in the terminal.
All module examples use these two variables — set them now and you will not need to copy your key again:

[source,bash,subs="attributes+",role="execute"]
----
export LITELLM_API_BASE="{litellm_api_base_url}"
export LITELLM_API_KEY="{litellm_virtual_key}"
----

== Step 3: Verify your connection

Confirm your key is active and list the models available to you:

[source,bash,role="execute"]
----
curl $LITELLM_API_BASE/models \
  -H "Authorization: Bearer $LITELLM_API_KEY" \
  -s | python3 -c "import sys,json; [print(m['id']) for m in json.load(sys.stdin)['data']]"
----

==== Verify

Expected output — the models your key can access:

[subs="attributes+"]
----
{litellm_available_models_list}
----

If you see model names printed, your connection is working correctly.

NOTE: If your key includes *Gemini 2.5 Pro*, always set `max_tokens` to at least *1000* in your requests.
Gemini 2.5 Pro is a thinking model that uses reasoning tokens before generating output — low token budgets return an empty response.
